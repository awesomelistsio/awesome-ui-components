## Component Libraries

- [Radix UI](https://www.radix-ui.com/) – Unstyled, accessible components for building high-quality design systems and web apps.
- [ShadCN UI](https://ui.shadcn.dev/) – Beautifully styled components built with Radix and Tailwind CSS.
- [Material UI (MUI)](https://mui.com/) – React components that implement Google's Material Design.
- [Chakra UI](https://chakra-ui.com/) – Simple, modular, accessible React components.
- [Ant Design](https://ant.design/) – A comprehensive enterprise-level UI design language and React component library.
- [Bootstrap](https://getbootstrap.com/) – The world’s most popular framework for building responsive, mobile-first websites.
- [Tailwind UI](https://tailwindui.com/) – Official UI components built with Tailwind CSS (paid).
- [Vuetify](https://vuetifyjs.com/) – Material Design UI library for Vue.js.
- [Quasar](https://quasar.dev/) – High-performance Material Design 2 UI for Vue 3.

## Design Systems

- [Carbon Design System](https://carbondesignsystem.com/) – IBM’s open-source design system.
- [Fluent UI](https://developer.microsoft.com/en-us/fluentui) – Microsoft’s official UI framework for building web apps.
- [Shopify Polaris](https://polaris.shopify.com/) – Shopify's design system for internal and external apps.
- [Lightning Design System](https://www.lightningdesignsystem.com/) – Salesforce’s UI framework and design system.

## Headless UI

- [Headless UI](https://headlessui.com/) – Completely unstyled, fully accessible UI components (by Tailwind Labs).
- [Ark UI](https://ark-ui.com/) – A headless library built on top of state machines, accessible by default.
- [React Aria](https://react-spectrum.adobe.com/react-aria/) – Adobe’s set of accessible UI primitives for building design systems.

## Accessibility-Focused Components

- [Reach UI](https://reach.tech/) – Accessible UI primitives for React.
- [React A11y Dialog](https://github.com/KittyGiraudel/react-a11y-dialog) – A React component wrapping the a11y-dialog library.
- [Accessible Rich Internet Applications (WAI-ARIA)](https://www.w3.org/WAI/standards-guidelines/aria/) – Guidelines and best practices for accessibility.

## Animation & Motion

- [Framer Motion](https://www.framer.com/motion/) – An animation library for React components.
- [React Spring](https://react-spring.dev/) – A spring-physics-based animation library for React.
- [Lottie](https://airbnb.io/lottie/#/) – Render After Effects animations natively on the web and mobile.

## Icon Sets

- [Lucide](https://lucide.dev/) – Beautiful & consistent icon toolkit (a fork of Feather icons).
- [Heroicons](https://heroicons.com/) – Hand-crafted icons by the creators of Tailwind CSS.
- [Font Awesome](https://fontawesome.com/) – The web’s most popular icon set and toolkit.
- [Remix Icon](https://remixicon.com/) – Open-source set of carefully crafted system icons.
- [Tabler Icons](https://tabler.io/icons) – A set of over 4,000 customizable SVG icons.

## Themes & Skins

- [DaisyUI](https://daisyui.com/) – Tailwind CSS component library with theming support.
- [Theme UI](https://theme-ui.com/) – Build consistent, themeable React apps based on design constraints.
- [Styled System](https://styled-system.com/) – Style props for rapid UI development.

## Component Showcases & Tools

- [Storybook](https://storybook.js.org/) – Develop, test, and document UI components in isolation.
- [Bit](https://bit.dev/) – Build and share reusable components across projects.
- [Backlight](https://backlight.dev/) – Design system platform for documenting and collaborating on UI components.
- [React Styleguidist](https://react-styleguidist.js.org/) – Isolated React component development environment.
